## 2. 수수료 AI 검증 관련 핵심 논의

### 2-1. 문제 인식

- 수수료는 단순 곱셈 구조가 아니라, 여러 항목을 합산하고 예외를 반영하는 복합 계산 구조다.
- 연도별 기준이 다르고, 규정집도 방대하며, HWP 문서 안에 텍스트와 표, 사례가 섞여 있어 전처리 난이도가 높다.
- 같은 수수료 항목이라도 설명 위치가 문서 내에서 분산되어 있어, AI가 바로 규정을 이해하기 어렵다.

### 2-2. 현실적인 방향성

- 생성형 AI를 최종 계산기나 최종 판정기로 쓰는 방식은 적합하지 않다는 공감대가 형성됐다.
- 현실적인 구조는 `결정론적 계산/검증 엔진 + AI 설명/검색/검토 보조 + 사람 승인`이다.
- 가장 먼저 풀어야 할 과제는 규정집 전체를 한 번에 완벽히 이해시키는 것이 아니라, 검증 가능한 기준 체계와 데이터 구조를 만드는 것이다.

### 2-3. 핵심 시사점

- AI는 수수료를 "맞힌다"기보다, 규정 검색, 근거 설명, 예외 후보 탐지, 리뷰 우선순위화에 더 적합하다.
- 수수료 금액, 환수, 보류, 최종 지급 여부 같은 최종 판단은 사람이 책임지는 구조가 필요하다.
- 규정 문서 전처리는 장기 과제이고, 단기적으로는 검증 가능한 범위부터 쪼개서 시작하는 것이 현실적이다.

## 3. 채널물량 인사이트 AI AGENT 관련 핵심 논의

### 3-1. AS-IS

현재는 물량이나 시나리오 판단이 필요할 때, 담당자가 관련 내용을 인적 데이터 기반으로 수집·확인한 뒤 수작업 판단하는 방식으로 운영된다.

대표적인 현재 업무 흐름은 다음과 같다.

1. 특정 월의 물량, 채널 흐름, 과거 이슈를 확인해야 한다.
2. 엑셀 파일, 회의자료, 시책 문서, 과거 메모를 각각 따로 찾는다.
3. 맥락이 부족하면 인적 데이터 기반으로 추가 확인한다.
4. 담당자가 숫자와 맥락을 다시 정리해 보고자료나 내부 판단 자료로 만든다.

### 3-2. TO-BE

목표 상태는 다음과 같다.

1. 담당자가 먼저 AGENT에 조회한다.
2. AGENT가 과거 물량, 이슈, 시책, 경쟁보험사 흐름을 함께 보여준다.
3. 다음달 총계 숫자와 채널별 방향성, 근거를 함께 제시한다.
4. 사람은 검토, 수정, 보고에 집중한다.

### 3-3. AGENT의 핵심 역할

- 조회형 역할
  - 특정 월의 업적 물량, 조직 효율, 시장 규모, 경쟁보험사 비교를 빠르게 조회
  - 관련 이슈, 시책, 신상품, 운영 배경을 함께 설명
- 예측형 역할
  - 다음달 총계 물량을 웬만하면 숫자로 제시
  - 전월 대비 방향성과 채널별 설명을 함께 제시
  - 임원 보고용 참고자료가 될 수 있도록 근거를 구조화
- 보고 보조 역할
  - 사람이 정리·보고하기 쉬운 문장 초안과 체크 포인트를 제공

### 3-4. 중요 원칙

- AI AGENT는 보조 도구다.
- 최종 판단과 공식 보고는 사람이 한다.
- 예측 숫자가 나오더라도, 그것은 참고용 인사이트이며 사람 검토가 선행돼야 한다.
- 설명은 반드시 근거 중심이어야 하며, 부족한 데이터는 부족하다고 밝혀야 한다.

## 4. 입력 데이터에 대한 합의

### 4-1. 숫자형 입력

- 월별 업적 물량
- 조직 효율 숫자
- CSM
- 시장 규모
- 경쟁보험사 숫자 및 비교 지표

### 4-2. 문서형 입력

- 월별 시책 세부내용
- 신상품/운영 공지
- 채널별 특이 이슈 메모

### 4-3. 운영 입력

- 담당자가 직접 넣는 월별 이슈
- 기존 보고자료 업로드

### 4-4. 입력 원칙

- 사람이 입력할 수 있는 범위만 요구한다.
- 시책효과 강도처럼 사전에 알 수 없는 값은 사람이 직접 수치화해서 넣지 않는다.
- AI는 과거 패턴과 동반 변수들을 바탕으로 시책효과를 추정하는 보조 역할을 맡는다.

## 5. 기대 출력에 대한 합의

### 5-1. 조회형 질문 예시

- `25년 4월 업적이 얼마였지?`
- `그 달에는 어떤 시책이 있었지?`
- `경쟁보험사 대비 이번달은 우세한 편이야?`

### 5-2. 예측형 질문 예시

- `이달 신상품이 나오고 시책을 전개하면 다음달 물량이 어떻게 될 것 같아?`
- `채널 A와 채널 B 중 어디가 더 증가할 가능성이 높아?`
- `경쟁보험사와 비교했을 때 다음달 우리가 어느 정도 우세할 것 같아?`

### 5-3. 예측 출력 원칙

- 총계 기준 숫자 예측이 가능하면 제시한다.
- 전월 대비 방향성이 맞아야 한다.
- 채널별 설명이 논리적으로 따라와야 한다.
- 예측의 근거와 추가 확인 포인트를 함께 제시한다.

## 7. 문구 및 디자인 수정 이력

오늘 문서와 이미지에 아래 수정사항을 반영했다.

- `채널장` 표현을 `유관부서 파트장`으로 변경
- 대상 사용자 중 불필요한 항목 삭제
- `월말 착지`를 `월말 마감`으로 변경
- 경쟁보험사 숫자 및 상대 우세/열세 비교 관점을 문서 전반에 추가
- AS-IS 표현을 `아는 사람에게 전화문의` 중심에서 `인적 데이터 기반 수집·확인 후 수작업 판단` 구조로 수정
- 임원보고용 이미지에서 `입력데이터` 영역을 정리하고, 하단 `핵심 전환` 메시지를 차분한 톤으로 강조
- 최종 이미지에서는 `시책 강도` 관련 별도 안내문을 제거
