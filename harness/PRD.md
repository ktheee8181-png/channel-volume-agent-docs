# 채널물량 인사이트 AI 에이전트 PRD v0.1

작성일: 2026-06-12  
문서 상태: 초안  
문서 목적: 삼성생명 영업/채널 담당자가 월별 업적 물량과 관련 이슈를 더 빠르게 조회하고, 다음달 물량을 근거 기반으로 예측하여 사람의 정리·보고를 돕는 AI 에이전트의 첫 기획 기준을 정리한다.

---

## Section 1. ICP / 주요 사용자 / 페르소나

### ICP
- 삼성생명 또는 유사한 보험 영업조직에서 채널별 물량을 관리하는 영업/운영 담당 조직
- 월별 업적 물량, 조직 효율, 시책, 신상품, CSM, 시장 규모 등 여러 데이터를 동시에 보고 판단해야 하는 조직
- 임원/채널장 보고를 위해 숫자와 해석을 함께 빠르게 정리해야 하는 조직

### 주요 사용자
1. 영업/채널 물량 담당자
2. 채널장 또는 팀장
3. 보고 자료를 만드는 실무 담당자

### 페르소나 A. 채널 물량 담당자
- 주간/월간으로 채널별 실적과 이슈를 확인한다
- 엑셀 수치와 회의자료/시책 문서를 함께 보며 해석한다
- 원하는 결과:
  - “25년 4월 물량이 얼마였지?” 같은 즉답형 조회
  - “신상품 + 시책 전개 시 다음달 물량이 어떻게 보이냐?” 같은 근거형 예측

### 페르소나 B. 보고 담당 실무자
- 임원/채널장 보고용 문장을 만들어야 한다
- 숫자만이 아니라 채널별 이슈와 방향성을 설명해야 한다
- 원하는 결과:
  - 총계 전망 숫자
  - 채널별 증감 방향
  - 그렇게 본 근거와 확인 포인트

### 페르소나 C. 채널장
- 세부 원자료를 다 읽을 시간은 없다
- 보고 숫자와 핵심 원인, 확인할 채널만 빠르게 보고 싶다

## Section 3. 문제 정의와 제공 가치

### 현재 문제
1. 숫자 조회는 가능하지만 숫자의 의미를 붙이는 데 시간이 오래 걸린다
2. 월별 시책/신상품/영업일수/시장 변화가 물량에 어떤 영향을 줬는지 일관되게 설명하기 어렵다
3. 다음달 물량 예측이 사람 경험에 크게 의존한다
4. 보고용 설명이 사람마다 달라 일관성이 떨어진다

### 제공 가치
1. 과거 실적/효율/시책/시장 데이터를 하나의 질의 인터페이스로 통합 조회
2. 다음달 총계 물량 전망을 숫자로 제시
3. 채널별로 방향성과 근거를 설명
4. 시책 효과를 정량·정성 혼합 방식으로 추정
5. 사람이 보고용 문장과 점검 포인트를 빠르게 정리할 수 있게 지원

### 핵심 가치 우선순위
1. 이슈 설명과 점검 포인트
2. 보고/브리핑 속도 개선
3. 대응 실행 참고자료 제공
4. 예측 정확도 향상

단, 사용자 기대상 “웬만하면 숫자 예측이 나와야 한다”는 요구가 있으므로 설명형만으로 끝내지 않고 예측형을 포함한다.

## Section 4. 결정 옵션 및 권장 방향

### 옵션 A. 조회형 에이전트만 구축
- 장점: 구현이 가장 빠름
- 단점: 사용자가 원하는 핵심 가치인 예측/인사이트가 약함

### 옵션 B. 예측형 에이전트만 구축
- 장점: 임팩트가 커 보임
- 단점: 데이터 구조화와 근거 설명이 약하면 신뢰를 잃기 쉬움

### 옵션 C. 조회형 + 예측형 통합 에이전트 구축
- 장점: 실제 사용 시나리오와 가장 잘 맞음
- 단점: 데이터 파이프라인과 답변 정책이 더 정교해야 함

### 권장 방향
- 최종 선택: 옵션 C
- 단, 출시 순서는 단계형으로 간다
  - Step 1: 조회 신뢰도 확보
  - Step 2: 설명형 인사이트 추가
  - Step 3: 총계 숫자 예측 + 채널별 방향성 예측 추가

## Section 5. Out of Scope

이번 버전에서 의식적으로 하지 않을 것:

1. 임원 기대치에 맞춘 보고용 숫자 보정
2. 최종 경영 목표 수치 산정 대행
3. 영업현장 지시문 자동 발행
4. 시책 효과의 인과관계를 100% 확정적으로 단정
5. 메신저/메일/보고서 전부 자동 수집하는 완전 자동형 운영
6. 채널별 숫자까지 모두 ±3~5% 정밀 예측하겠다는 약속
7. 사람 검토 없이 배포되는 공식 전망 확정 기능

## Section 7. 에이전트 역할 / Primary Goal / Anti-Goals

### Role
- 채널 물량과 관련 이슈를 통합 해석하는 조회·예측 보조 에이전트

### Primary Goal
- 총계 기준 다음달 물량을 가능한 실제 착지에 가깝게 예측하고, 그 근거를 사람이 보고에 쓸 수 있는 형태로 설명한다

### Secondary Goals
1. 과거 데이터 조회 시간을 줄인다
2. 채널별 증감 방향과 이슈를 빠르게 파악하게 한다
3. 보고용 서술 초안을 일관되게 제공한다

### Anti-Goals
1. 임원 입맛에 맞는 숫자를 만드는 도구가 되지 않는다
2. 근거가 약한데도 특정 시책 효과를 단정하지 않는다
3. 최종 경영판단이나 영업 지시를 대신하지 않는다
4. 불충분한 데이터인데도 자신감 있게 확정 수치를 말하지 않는다
5. 입력 원천값을 임의로 수정하지 않는다

## Section 8. 입력 데이터 / 도구 / 연동

### 핵심 입력 데이터
- 업적 물량
- 조직 효율 숫자
- 시책 세부내용
- 신상품 정보
- 영업일수
- CSM
- 시장 규모
- 채널별 특이사항

### 입력 방식
1. 운영자 직접 입력
2. 기존 보고서/자료 업로드

### 운영 원칙
- 시책 강도는 사람이 사전 수치화하지 않는다
- AI가 과거 패턴과 동반 변수 변화를 함께 보고 효과를 추정한다
- 최근 1~2년 흐름을 더 강하게 보되, 변수에 따라 장기 패턴도 참고한다

### 추후 필요한 연동 후보
- 엑셀 업로드
- 문서 업로드
- 저장소/DB
- 대시보드

## Section 9. 컨텍스트 / 메모리 설계

### Working Context
- 최근 월/주/일 데이터
- 현재 월 진행 상황
- 최신 시책/신상품/영업일수 정보

### Long-term Context
- 최근 5년 이력
- 채널별 계절성
- 과거 유사 시책/신상품 패턴

### Context Weighting
- 기본 원칙: 최근 1~2년 가중치를 더 높게 둔다
- 단, 변수별로 장기 패턴 참고 여부를 분리한다
  - 계절성/영업일수: 장기 패턴 가치 높음
  - 최근 채널 분위기/시책 반응: 최근성 가치 높음

## Section 10. Trigger / 실행 흐름

### Trigger
1. 수동 질의
2. 주간/수시 데이터 업데이트
3. 월말/월초 전망 갱신

### 실행 흐름
1. 최신 데이터 로드
2. 질의 유형 분류
   - 조회형
   - 예측형
   - 설명형
3. 관련 수치/문서/이슈 검색
4. 총계 전망 계산
5. 채널별 방향성 및 근거 생성
6. 보고용 답변 구성

### 질의 유형 예시
- 조회형: `25.4월 업적이 얼마였지?`
- 예측형: `이달 신상품 출시와 시책 전개가 있으면 다음달 물량이 어떻게 될까?`
- 설명형: `왜 이번달 채널 A 물량이 둔화됐다고 보는지 설명해줘`

---

## Section 11. Output Specification

### 기본 답변 원칙
- 숫자 예측은 웬만하면 제공한다
- 단일 숫자만 던지지 않고 근거를 함께 제공한다
- 채널별로는 방향성과 이슈를 설명한다
- 사람이 보고용으로 바로 편집 가능한 문장 형태를 지원한다

### 조회형 출력 예시
- `2025년 4월 총 업적 물량은 XXX였습니다. 주요 기여 채널은 A, B였고, 전월 대비 증감률은 YY%였습니다.`

### 예측형 출력 예시
- `다음달 총계 물량 전망은 98~102 수준으로 봅니다. 중심 전망은 100입니다.`
- `근거: 최근 2개월 추세 유지, 신상품 출시 효과 반영, 시책 전개 확대, 영업일수 +1일`
- `채널별 방향성:`
  - `채널 A: 증가 가능성 높음. 신상품 반응과 시책 연계 효과 예상`
  - `채널 B: 보합 또는 소폭 감소 가능성. 직전 반등 이후 기저효과와 운영 이슈 존재`
- `확인 포인트: 채널 A 초기 반응, 채널 B 시책 집행 시점, 영업일수 효과 실현 여부`

### 보고용 출력 예시
- `다음달 총계 물량은 전월 대비 소폭 증가 전망입니다. 최근 추세와 신상품 출시, 시책 전개 효과가 반영될 것으로 보입니다. 다만 채널별로는 편차가 예상되며, 채널 A는 확대 가능성이, 채널 B는 보합 가능성이 상대적으로 높습니다.`

## Section 12. 성공 지표

### 핵심 성과 지표
1. 총계 물량 예측 오차: ±3~5% 이내
2. 전월 대비 방향성 적중률
3. 채널별 설명 수용도
4. 보고자료 작성 시간 절감

### 운영 지표
1. 질의 응답 시간
2. 데이터 최신화 주기
3. 입력 누락률
4. 예측 갱신 빈도
5. 사용자 재사용률

### 품질 지표
1. 사용자가 “근거가 납득된다”고 평가한 비율
2. 예측 후 사후분석에서 주요 원인 설명이 유지되는 비율

## Section 13. 핵심 가정과 검증 과제

### H1. 데이터 품질 가정
- 월별/주별/일별 + 채널별 데이터가 충분히 정리되어 있다
- 검증 방법:
  - 최근 12개월 데이터 샘플 점검
  - 누락/형식 불일치 비율 측정

### H2. 시책 효과 추정 가정
- 시책 강도를 사전 수치화하지 않아도, 동반 변수와 과거 패턴을 통해 효과를 유의미하게 추정할 수 있다
- 검증 방법:
  - 과거 시책 월을 뽑아 사후 설명 적합도 확인

### H3. 출력 활용 가정
- 사용자는 AI 전망을 최종 확정치가 아니라 참고·정리·보고용으로 사용할 것이다
- 검증 방법:
  - 실사용자 인터뷰
  - 실제 보고자료 반영 사례 수집

---

## Section 14. 실패 모드 / HITL

### 실패 모드
1. 최신 데이터가 누락되었는데도 예측을 수행함
2. 시책 효과를 다른 변수 영향과 혼동함
3. 채널별 설명은 그럴듯하지만 총계 숫자가 크게 틀림
4. 설명은 맞는데 수시 갱신 타이밍이 늦음
5. 임원 기대 숫자와 다르다는 이유로 신뢰를 잃음

### 대응 원칙
1. 데이터 누락 시 경고 표시
2. 근거 요인별 기여 설명 분리
3. 최근성과 장기패턴 가중치를 분리 적용
4. 총계 전망과 채널 방향성을 별도 관리
5. “가장 가능성 높은 실제 착지 전망”이라는 역할을 명확히 유지

### Human-in-the-loop
- 최종 보고 전 사람 검토 필수
- 중요한 숫자 변경 시 재검토 유도
- 근거가 약한 경우 “추가 확인 필요” 표시

---

## Section 15. QA Pool / 검증 시나리오

### 기본 테스트 시나리오
1. 과거 월 조회 정확도 테스트
2. 특정 시책 월 사후 설명 테스트
3. 월초 예측 vs 월말 실제치 비교 테스트
4. 채널별 방향성 적중 테스트
5. 동일 질의 반복 시 일관성 테스트

### 적대적 질문 예시
1. `근거가 약한데도 왜 증가라고 봤지?`
2. `이 수치는 신상품 때문이 아니라 영업일수 때문 아닌가?`
3. `최근 2년만 보면 감소인데 왜 장기 패턴을 반영했지?`
4. `채널 B 전망이 보수적인데 어떤 데이터를 봤나?`

### 첫 파일럿 기준
- 특정 1개 연도 + 1개 대분류/채널군
- 최근 데이터 중심으로 학습/검증
- 총계 예측 정확도와 채널 설명력을 우선 검증

## 부록. 첫 버전 핵심 제품 원칙

1. 하나의 에이전트에서 조회와 예측을 모두 지원한다
2. 총계는 숫자 예측을 제공한다
3. 채널별로는 방향성과 근거 설명을 제공한다
4. 시책효과는 AI가 해석하되, 사람은 원천 데이터만 입력한다
5. 결과는 사람의 참고·정리·보고용이다
6. 임원 기대를 맞추는 숫자보다 실제 착지 가능성이 높은 숫자를 우선한다
