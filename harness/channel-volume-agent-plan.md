# 채널물량 인사이트 AI AGENT 기획서

작성일: 2026-06-12  
문서 목적: 삼성생명 영업/채널 조직의 물량 조회, 이슈 해석, 시나리오 기반 물량 예측 업무를 지원하는 `채널물량 인사이트 AI AGENT`의 상세 기획 기준을 정리한다.

---

## AS-IS

현재는 물량이나 시나리오 판단이 필요할 때, 담당자가 관련 내용을 가장 잘 아는 사람에게 직접 전화하거나 메신저로 문의하면서 판단하는 방식으로 운영된다.

대표적인 현재 업무 흐름은 아래와 같다.

1. 담당자가 특정 월의 물량, 채널 흐름, 과거 이슈를 확인해야 한다.
2. 엑셀 파일, 회의자료, 시책 문서, 실무자 기억을 각각 따로 찾아본다.
3. 그래도 맥락이 부족하면 “이때 물량 얼마로 봤지?”, “그때 무슨 이슈 있었지?”, “다음달은 어느 정도로 예상해?” 같은 질문을 유선 또는 메신저로 물어본다.
4. 답변을 받은 뒤 담당자가 숫자와 맥락을 다시 정리해서 보고자료나 내부 판단 자료로 만든다.

이 방식의 특징은 다음과 같다.

- 데이터는 존재하지만, 사람의 머릿속에 있는 해석과 맥락 의존도가 높다.
- 숫자 조회와 이슈 설명과 예측 판단이 분리되지 않고 한 사람의 경험에 묶여 있다.
- 같은 질문에 대해서도 누구에게 묻느냐에 따라 답변이 달라질 수 있다.
- 임원 보고용 숫자와 근거를 만들 때 시간이 오래 걸리고 방어 논리가 약해질 수 있다.
- 담당자가 바뀌거나 부재 중이면 판단 품질과 속도가 급격히 떨어질 수 있다.

## 1. 문제정의

### 1-1. 핵심 문제

채널 물량 관련 데이터는 숫자 형태로 많이 축적되어 있으나, 월별 이슈, 시책, 신상품, 영업일수, 시장 변화와 같은 해석 요소가 문서와 사람의 기억에 분산되어 있어, 조회와 예측과 설명이 표준화되지 못하고 있다.

### 1-2. 세부 문제

#### 문제 A. 조회는 가능하지만, 해석까지 연결되지 않는다

- 월별 업적 물량, 조직 효율, 시장 규모 등의 숫자는 엑셀이나 자료로 존재한다.
- 하지만 “왜 그 숫자가 나왔는지”, “어떤 시책이나 이슈가 영향을 줬는지”는 자료가 흩어져 있거나 사람 기억에 남아 있다.
- 따라서 단순 숫자 조회 이후에도 추가 설명을 위해 다시 사람에게 물어봐야 한다.

#### 문제 B. 예측이 사람 경험과 네트워크에 과도하게 의존한다

- 다음달 물량을 전망할 때 체계적 모델보다 경험 많은 담당자의 감각과 과거 기억이 크게 작용한다.
- 같은 데이터를 두고도 사람마다 다른 전망을 할 수 있다.
- 예측 근거가 구조화되지 않아 사후 검증과 개선이 어렵다.

#### 문제 C. 시책 효과를 체계적으로 누적 학습하지 못한다

- 시책 세부내용은 존재하지만, 시책이 실제로 어느 정도 물량 변화에 영향을 줬는지를 조직 차원에서 정형화해 축적하지 못하고 있다.
- 결국 매번 비슷한 질문을 반복하면서도, 학습 자산이 쌓이지 않는다.

#### 문제 D. 보고자료 작성이 반복적이고 비효율적이다

- 실무자는 숫자를 찾고, 과거 맥락을 수집하고, 채널별 이슈를 정리하고, 전망 숫자와 설명을 조합해 보고 문장을 만들어야 한다.
- 이 과정이 모두 수작업이라 시간이 오래 걸리고 편차가 발생한다.

#### 문제 E. 지식이 사람에게 잠겨 있다

- “그 시기에 어떤 이슈가 있었는지”, “이 채널은 이런 반응을 보였는지”, “이 시책은 실제로 먹혔는지” 같은 정보가 특정 담당자에게 집중돼 있다.
- 담당자 부재, 이동, 조직 개편 시 지식 단절 위험이 크다.

### 1-3. 해결해야 하는 본질

이 AGENT가 해결하려는 본질은 `숫자 데이터`와 `맥락 데이터`와 `사람의 경험적 판단`을 하나의 조회·설명·예측 체계로 연결하는 것이다.

### 1-4. 목표 상태

- 사람이 누군가에게 전화해서 묻지 않아도, 과거 물량과 이슈를 빠르게 확인할 수 있어야 한다.
- “왜 이렇게 보는지”에 대한 근거가 구조화된 형태로 제공되어야 한다.
- 다음달 총계 물량은 웬만하면 숫자로 예측되어야 한다.
- 채널별로는 방향성과 주요 이슈가 설명되어야 한다.
- 최종 숫자 확정과 보고 판단은 사람이 하되, AI가 검토와 정리 시간을 줄여줘야 한다.

## 2. 대상사용자

### 2-1. 1차 대상 사용자

#### 사용자 A. 채널 물량 담당자

- 월별/주별/일별 업적과 채널 흐름을 실무적으로 관리하는 담당자
- 주로 확인하는 질문:
  - `25.4월 업적이 얼마였지?`
  - `작년 같은 시기랑 비교하면 어때?`
  - `이달 시책 전개 시 다음달 물량이 어느 정도 될까?`
  - `왜 채널 A는 증가, 채널 B는 둔화로 보는 거지?`
- 기대 가치:
  - 빠른 조회
  - 예측 초안 확보
  - 설명 논리 확보
  - 점검 포인트 정리

#### 사용자 B. 보고자료 작성 실무자

- 임원/채널장 보고자료를 만드는 사람
- 필요한 산출물:
  - 총계 전망 숫자
  - 전월 대비 방향성
  - 채널별 이슈 요약
  - 보고용 문장 초안
- 기대 가치:
  - 자료 조합 시간 단축
  - 설명 일관성 확보
  - 보고 문장 초안 자동화

### 2-2. 2차 대상 사용자

#### 사용자 C. 유관부서 파트장

- 세부 데이터는 다 보지 않지만 핵심 숫자와 이유를 빠르게 알아야 하는 의사결정자
- 기대 가치:
  - 총계 전망 숫자
  - 채널별 방향성
  - 체크가 필요한 포인트
  - 경쟁보험사 대비 상대 우세/열세 흐름 파악

#### 사용자 D. 조직 내 유관 실무자

- 시책, 상품, 영업 운영과 관련된 실무자
- 기대 가치:
  - 시책 반응 해석
  - 과거 유사 사례 참고
  - 물량 변화 원인 추적

## 3. 핵심기능 1: 조회형 인사이트 기능

### 3-1. 기능 정의

사용자가 자연어로 질문하면, 과거 물량/효율/시장/시책/이슈 데이터를 연결해 필요한 숫자와 맥락을 즉시 조회해주는 기능이다.

### 3-2. 왜 필요한가

현재는 숫자 조회와 맥락 확인이 각각 다른 파일과 사람에게 흩어져 있어, 단순 질문 하나에도 여러 자료를 뒤져야 한다. 이 기능은 가장 기본적인 `찾기` 문제를 해결한다.

### 3-3. 주요 질문 유형

#### 유형 A. 숫자 조회

- `25년 4월 총 업적 물량이 얼마였지?`
- `채널 A의 3월 물량은 전월 대비 얼마나 늘었지?`
- `작년 동월 대비 실적은 어땠지?`

#### 유형 B. 비교 조회

- `이번달이 작년 같은 시기보다 좋은 편이야?`
- `최근 3개월 중 어떤 채널이 가장 흔들렸지?`
- `신상품 출시 전후로 채널별 흐름이 달라졌나?`
- `경쟁보험사 대비 이번달은 우세한 편이야?`
- `경쟁보험사와 비교했을 때 어떤 채널이 상대적으로 강했지?`

#### 유형 C. 맥락 조회

- `이때 무슨 이슈가 있었지?`
- `그 달에는 어떤 시책이 있었지?`
- `영업일수나 시장 변수는 어땠지?`

#### 유형 D. 설명 조회

- `왜 그 달 물량이 많이 들어왔다고 보는 거야?`
- `왜 채널 B는 반등이라고 보는 거야?`

### 3-4. 세부 동작

1. 사용자의 질문이 숫자 조회인지, 비교인지, 맥락 설명인지 분류한다.
2. 관련 기간, 채널, 변수, 시책 정보와 경쟁보험사 비교 데이터를 찾는다.
3. 숫자 데이터와 월별 이슈/시책 내용을 함께 묶어 보여준다.
4. 필요한 경우 전월/전년동월/최근 추세와 경쟁보험사 대비 흐름을 함께 요약한다.
5. 사용자가 바로 보고에 쓸 수 있도록 짧은 설명 문장을 함께 제공한다.

### 3-5. 사용자에게 보여줄 답변 구성

- 핵심 숫자
- 비교 기준
- 관련 이슈
- 짧은 해석
- 경쟁보험사 대비 상대 위치
- 추가로 볼만한 연관 지표

### 3-6. 예시 출력

`2025년 4월 총 업적 물량은 XXX였습니다. 전월 대비 YY% 증가했고, 작년 동월 대비 ZZ% 증가했습니다. 주요 배경으로는 신상품 출시, 시책 전개 강화, 영업일수 증가가 확인됩니다. 채널 A가 증가를 주도했고 채널 B는 보합 수준이었습니다. 경쟁보험사 대비로는 채널 A에서 상대적으로 우세한 흐름이 확인됩니다.`

### 3-7. 이 기능의 성공 기준

- 숫자 조회 정확도가 높다
- 관련 이슈를 빠르게 연결한다
- 사용자가 사람에게 추가로 전화하는 빈도를 줄인다

## 4. 핵심기능 2: 예측형 인사이트 기능

### 4-1. 기능 정의

사용자가 신상품, 시책, 최근 흐름, 영업일수, 시장 상황 등을 바탕으로 다음달 물량을 물어보면, AGENT가 총계 기준의 숫자 예측과 채널별 방향성, 예측 근거, 확인 포인트를 제공하는 기능이다.

### 4-2. 왜 필요한가

현재는 다음달 물량 예측이 경험 많은 담당자에게 의존하며, 그 근거도 구두로만 전달되는 경우가 많다. 이 기능은 예측을 구조화하고, 근거와 함께 재사용 가능한 형태로 남기는 역할을 한다.

### 4-3. 이 기능이 다루는 예측 질문

- `이달 신상품이 나오고 시책을 전개하면 다음달 물량이 어떻게 될 것 같아?`
- `최근 2주 흐름과 영업일수를 보면 월말 마감은 어느 정도로 봐야 해?`
- `채널 A와 B 중 어느 쪽이 다음달 반등 가능성이 높아?`
- `이 시책은 실제로 물량 상승에 도움이 될 것 같아?`
- `경쟁보험사와 비교했을 때 다음달 우리가 어느 정도 우세할 것 같아?`

### 4-4. 예측 방식의 기본 철학

- 임원 기대치를 맞추는 숫자가 아니라 `가장 가능성 높은 실제 착지 전망`을 제시한다.
- 총계 물량은 웬만하면 숫자로 예측한다.
- 채널별로는 정밀 수치보다 방향성과 근거 설명을 우선한다.
- 예측 결과는 사람이 참고·정리·보고하는 용도이며, 최종 확정치는 사람이 판단한다.

### 4-5. 예측 시 고려할 입력 요인

- 최근 1~2년 추세
- 과거 5년 패턴
- 월별/주별/일별 물량 흐름
- 영업일수
- 시책 세부내용
- 신상품 이슈
- 시장 규모
- 조직 효율
- 채널별 특이사항
- 경쟁보험사 숫자 및 흐름

### 4-6. 예측 출력 방식

#### 총계 예측

- 다음달 총계 전망 숫자
- 가능 범위
- 기준 시나리오

#### 채널별 인사이트

- 증가/보합/감소 방향
- 주요 이슈
- 시책 반응 예상
- 확인이 필요한 변수

#### 경쟁보험사 비교 인사이트

- 경쟁보험사 대비 상대 우세/열세
- 격차가 벌어진 원인 후보
- 자사 강점 채널과 열세 채널
- 다음달 경쟁 구도 전망

#### 설명 문장

- `왜 이렇게 봤는지`
- `어떤 요인이 가장 크게 작용했는지`
- `어떤 부분은 아직 불확실한지`

### 4-7. 예시 출력

`다음달 총계 물량은 98~102 수준으로 전망합니다. 중심 전망은 100입니다. 최근 2개월 흐름 유지, 신상품 출시 효과, 시책 전개 확대, 영업일수 증가가 반영된 결과입니다. 채널 A는 증가 가능성이 높고, 채널 B는 보합 또는 소폭 감소 가능성이 있습니다. 채널 A는 신상품 초기 반응이 좋고 시책 연계 효과가 예상되며, 채널 B는 직전 반등에 따른 기저효과와 운영 이슈가 있습니다. 경쟁보험사 대비로는 채널 A에서 상대 우세 가능성이 높고, 채널 B는 격차 축소 여부를 추가 확인해야 합니다. 추가 확인 포인트는 채널 A 초기 반응 지속 여부, 채널 B 시책 집행 시점입니다.`

### 4-8. 성공 기준

- 총계 기준 예측 오차가 실무 수용 수준에 들어온다
- 전월 대비 방향성이 맞는다
- 채널별 설명이 사용자에게 납득된다
- 사용자가 보고용 문장으로 쉽게 전환할 수 있다

이번 AGENT에서 의도적으로 만들지 않을 것들을 명확히 정의한다.

### 5-1. 하지 않을 기능

1. AI가 최종 경영 숫자를 확정하는 기능

4. 채널별 숫자까지 모두 매우 정밀하게 맞추겠다고 약속하는 기능
5. 모든 외부 자료를 자동으로 수집해 완전 자동으로 학습하는 기능
6. 시책 강도를 사람이 일일이 수치 입력해야만 동작하는 구조

8. 근거가 부족한데도 단정적으로 “이 시책 때문”이라고 결론 내리는 기능
9. 경쟁보험사 우위/열세를 근거 없이 단정하는 기능

### 5-2. 안 만드는 이유

- 조직 내 신뢰를 잃지 않기 위해서
- 설명 책임을 분명히 하기 위해서
- 예측과 판단을 구분하기 위해서
- 초기 도입 난이도를 낮추기 위해서
- 실무 입력 가능 범위 안에서 시작하기 위해서

## 7. 사람승인

### 7-1. 사람승인이 필요한 이유

이 AGENT는 인사이트와 예측을 제공하지만, 조직 내에서 실제 보고와 실행은 사람 책임으로 이뤄져야 한다. 특히 예측 숫자는 참고용이며, 공식적으로 공유되기 전에는 반드시 사람 검토가 필요하다.

### 7-2. 사람승인이 필요한 시점

#### 승인 시점 A. 공식 보고 전

- 임원 보고
- 채널장 공유
- 대외/대내 공식 자료 반영

#### 승인 시점 B. 예측 수치가 크게 바뀐 경우

- 전주 대비 전망이 급변한 경우
- 특정 채널 방향성이 반전된 경우

#### 승인 시점 C. 근거가 약한 경우

- 입력 데이터가 부족한 경우
- 시책 효과 추정 근거가 약한 경우
- 서로 상충하는 이슈가 많은 경우

### 7-3. 사람승인 프로세스

1. AGENT가 조회/예측 결과를 제시한다.
2. 담당자가 숫자와 근거를 검토한다.
3. 필요 시 추가 자료나 현장 확인을 붙인다.
4. 담당자가 보고용 문장을 수정/보완한다.
5. 최종적으로 사람이 승인한 버전만 공식 보고에 사용한다.

### 7-4. 사람이 꼭 확인해야 하는 항목

- 총계 전망 숫자
- 전월 대비 방향성
- 채널별 해석
- 근거로 사용된 주요 변수
- 최근 반영된 이슈가 누락되지 않았는지 여부
- 경쟁보험사 비교 기준이 동일한지 여부

### 7-5. 사람승인 이후 남겨야 하는 기록

- 최종 보고 숫자
- AGENT 원안 숫자
- 사람이 수정한 내용
- 수정 이유
- 최종 보고 문장

이 기록은 이후 예측 품질을 개선할 때 매우 중요한 학습 자산이 된다.
