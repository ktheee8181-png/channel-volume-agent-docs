# 채널물량 인사이트 AI 에이전트 PRD v0.2

## 문제 정의

- 삼성생명 영업/채널 담당자는 월별 업적 물량, 채널별 흐름, 상품 분류별 실적, 신상품, 시장/경쟁사 변수, 월별 이벤트를 따로 보고 해석해야 해서 조회, 설명, 예측, 보고 작성이 사람 경험에 크게 의존한다.
- 사용자는 보고가 필요한 시점마다 에이전트를 직접 실행해 현재까지의 수치와 이슈를 반영한 다음달 전망을 즉시 받고 싶어 한다.
- 이 제품은 `판단 불가`로 멈추지 않고 가능한 한 항상 전망을 제시하되, `예측 범위`, `신뢰도`, `근거`, `추가 확인 포인트`를 함께 제공해 사람의 정리와 보고를 돕는다.

## 대상 사용자

- 영업/채널 물량 담당자
- 보고자료 작성 실무자
- 유관부서 파트장

## 핵심 기능 1

- 자연어 기반 `조회 + 설명` 기능을 제공한다.
- 사용자는 특정 월의 총 물량, 채널별 증감, 상품 대분류/중분류별 흐름, 신상품 영향, 경쟁사/시장 변화, 월별 이벤트를 질문할 수 있다.
- 에이전트는 `monthly_summary`, `main_fact`, `monthly_events`, `special_products`를 함께 읽고 숫자와 이유를 한 번에 설명한다.

## 핵심 기능 2

- 사용자 요청 시점 기준 `온디맨드 예측 + 보고 초안 생성` 기능을 제공한다.
- 에이전트는 최근 5년 월별 수치, 채널 구조, 상품 구조, 신상품, 판촉비총량, 영업일수, 시장/경쟁사 지표, 월별 이벤트를 반영해 다음달 총계 전망을 항상 제시한다.
- 채널별로는 방향성과 주요 원인을 함께 설명하고, 보고용 문장 초안까지 생성한다.

## 입력 데이터

- 입력 데이터 기준은 `github_deploy_package/docs/dummy.md`의 구조를 따른다.

### 1. `monthly_summary.csv` / `monthly_summary` 시트

- 연도
- 월
- 월초
- 보장월초
- 건강월초
- 건강_순수형
- 건강_환급형
- 건강_특화형
- 종신월초
- 종신_목적종신
- 종신_일반종신
- 연금월초
- 연금_세부
- 저축_세부
- 시장전체
- `M/S`
- 경쟁사보장월초
- 영업일수
- 판촉비총량
- 채널합계검산
- 상품합계검산
- 합계차이

### 2. `main_fact.csv` / `main_fact` 시트

- 연도
- 월
- 채널
- 대분류
- 중분류
- 금액

### 3. `monthly_events.csv` / `monthly_events` 시트

- 월별 계절성
- 시장 이슈
- 경쟁사 특이사항
- 제도/환경 변화
- 시책/운영 이슈

### 4. `special_products.csv` / `special_products` 시트

- 월별 신상품 1개
- 별도 관리 상품
- 설명 필드
- 상품 중분류

### 5. 참조 데이터

- `channel_profile.csv`
- `product_profile.csv`

### 6. 데이터 무결성 규칙

- `월초 = 보장월초 + 연금월초`
- `보장월초 = 건강월초 + 종신월초`
- `건강월초 = 건강_순수형 + 건강_환급형 + 건강_특화형`
- `종신월초 = 종신_목적종신 + 종신_일반종신`
- `채널합계검산 = 상품합계검산`
- `합계차이 = 0`

### 7. 신상품 규칙

- 월별 신상품 1개
- 전체 기간 기준 총 53개
- 건강:종신 비율 약 3:1
- 건강/종신 혼합 행 금지

### 8. 보안 원칙

- 실제 고객정보, 계약정보, 사내 민감정보는 입력하지 않고 시연용 더미 데이터만 사용한다.

## 출력 형식

- 기본 출력은 한국어 브리핑형 텍스트다.
- 모든 응답에는 `질의 시점`, `사용 데이터 기준월`, `예측 대상월`을 포함한다.

### 조회형 출력

- 조회값
- 전월/전년 비교
- 채널별/상품별 기여
- 관련 이벤트
- 확인 포인트

### 설명형 출력

- 왜 증가 또는 감소로 보는지에 대한 주요 근거 3~5개

### 예측형 출력

- 중심 전망
- 예측 범위
- 신뢰도
- 주요 근거
- 채널별 방향성
- 보고용 문장 초안

### 출력 예시

`다음달 총계 물량 중심 전망은 100이며 예측 범위는 96~103입니다. 신뢰도는 중간입니다. 주요 근거는 최근 2개월 회복세, 신상품 출시, 영업일수 증가, 경쟁사 보장월초 둔화입니다. FC와 GFC는 증가 가능성이 높고, 디지털은 보합 가능성이 높습니다.`

## 사람 승인 지점(HITL)

- 공식 보고서 반영 전에는 사람이 최종 승인한다.
- 임원 보고, 목표 수치 해석, 실행 지시 전환은 반드시 사람 승인 후 진행한다.
- 신뢰도가 낮거나 최신 월 데이터가 덜 반영된 경우에는 재검토를 요청한다.
- 직전 전망 대비 변동 폭이 큰 경우에는 사람 확인을 거친다.
- 에이전트는 전망과 설명을 제공하지만 최종 확정 숫자와 최종 경영 판단은 대신하지 않는다.

## 완료 정의

- 사용자가 원하는 시점에 자연어로 조회, 설명, 예측을 실행할 수 있다.
- `monthly_summary`, `main_fact`, `monthly_events`, `special_products`를 함께 활용해 답변한다.
- 총계 전망은 항상 제공되고, 채널별 방향성도 함께 제공된다.
- 출력에 예측 범위, 신뢰도, 주요 근거, 데이터 기준 시점이 포함된다.
- 입력 데이터 무결성 규칙을 사전 검증한 뒤 예측에 사용한다.
- 공식 보고 전 사람 승인 흐름이 명시돼 있다.

## 보류 이유(한 줄)

- 채널·상품 단위의 고정밀 세분 예측과 공식 보고 자동 배포는 실제 운영 데이터 검증과 승인 규칙 정립 전까지 보류한다.

## 후보 검증 질문 5개

- 가장 약한 가정은? `monthly_summary + main_fact + monthly_events` 조합만으로 사용자가 납득할 수준의 근거형 예측이 가능한가?
- 데이터가 부족한 부분은? `monthly_events`가 채널별 현장 이슈와 시책 변화를 설명하기에 충분한가?
- 보안 위험은? 더미 단계는 안전하지만 실제 데이터 전환 시 개인정보·계약정보가 섞일 위험은 없는가?
- 비용 위험은? 사용자가 보고 싶을 때마다 돌리는 온디맨드 예측 구조에서 LLM 호출량과 보고 초안 생성 비용이 감당 가능한가?
- 사람 승인 지점은 충분한가? 공식 보고, 큰 전망 변동, 저신뢰 예측, 범위 밖 해석에 대해 사람 승인 규칙이 충분히 강한가?
