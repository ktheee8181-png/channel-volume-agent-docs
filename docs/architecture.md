# Architecture.md

## Goal

- 영업/채널 물량 담당자, 보고자료 작성 실무자, 유관부서 파트장이 원하는 시점에 채널물량 `조회`, `설명`, `예측`을 실행할 수 있게 한다.
- 에이전트는 총계 전망을 항상 제시하되, `신뢰도`, `근거`, `확인 포인트`를 함께 제공해 사람이 보고와 판단을 이어갈 수 있게 한다.
- 최종 확정 숫자와 최종 경영 판단은 사람이 가지도록 설계한다.

## Input

- 구조화 데이터
  - `monthly_summary`
  - `main_fact`
  - `special_products`
  - `channel_profile`
  - `product_profile`
- 반구조화 데이터
  - `monthly_events`
- 입력 기준
  - `monthly_summary`와 `main_fact`는 기준 수치 데이터다.
  - `monthly_events`와 `special_products`는 설명과 예측 근거 강화용 데이터다.
  - 모든 입력은 합계/검산 규칙을 먼저 통과해야 한다.

## Process

1. 사용자 질의를 수신한다.
2. 질의를 `조회형`, `설명형`, `예측형`으로 구분한다.
3. 구조화 데이터에서 기준 수치를 우선 조회한다.
4. `monthly_events`와 `special_products`에서 관련 이슈와 신상품 정보를 연결한다.
5. 조회형은 현재값과 비교값을 정리한다.
6. 설명형은 증감 원인과 관련 이벤트를 요약한다.
7. 예측형은 최근 흐름, 계절성, 신상품, 영업일수, 시장/경쟁사 변수를 반영해 총계 중심 전망과 채널별 방향성을 생성한다.
8. 결과를 `신뢰도`, `주요 근거`, `확인 포인트`와 함께 브리핑형 문장으로 출력한다.

## Tools

- 데이터 로더
  - CSV/시트 데이터를 읽어온다.
- 검산 로직
  - 합계 규칙과 무결성 규칙을 확인한다.
- 질의 분류기
  - 사용자의 질문을 조회/설명/예측 유형으로 나눈다.
- 근거 추출기
  - 관련 월, 채널, 상품, 이벤트, 신상품 정보를 묶어준다.
- 예측 엔진
  - 최근 흐름과 보조 변수를 반영해 중심 전망과 범위를 계산한다.
- 응답 생성기
  - 최종 결과를 한국어 브리핑 형식으로 정리한다.

## Output

- 조회형
  - 조회값
  - 전월/전년 비교
  - 채널별/상품별 기여
  - 관련 이벤트
- 설명형
  - 증감 이유
  - 주요 근거 3~5개
  - 확인 포인트
- 예측형
  - 중심 전망
  - 예측 범위
  - 신뢰도
  - 주요 근거
  - 채널별 방향성
  - 보고용 문장 초안
- 공통 메타정보
  - 질의 시점
  - 사용 데이터 기준월
  - 예측 대상월

## HITL

- 공식 보고 전에는 사람 승인이 필요하다.
- 신뢰도가 낮은 예측은 재검토 대상으로 표시한다.
- 직전 전망 대비 변동 폭이 큰 경우에는 사람 확인을 요청한다.
- 에이전트는 설명과 전망을 제공하지만 공식 확정 숫자와 실행 지시는 대신하지 않는다.

## Failure

- 입력 데이터 누락 또는 최신성 부족
  - 영향: 잘못된 설명 또는 저신뢰 예측
- 합계 검산 실패
  - 영향: 기준값 오류로 전체 답변 신뢰도 하락
- 이벤트 데이터 빈약
  - 영향: 설명 근거가 약해짐
- 예측 근거 부족
  - 영향: 전망은 제공하되 신뢰도 낮음으로 표시 필요
- 사람 승인 없이 공식 숫자로 사용
  - 영향: 운영 리스크 증가

## Cost

- 주요 비용 요소
  - 데이터 적재 및 조회 비용
  - LLM 호출 비용
  - 예측 실행 빈도 증가에 따른 처리 비용
- 비용 절감 원칙
  - 구조화 데이터 조회를 먼저 수행하고, LLM은 설명/브리핑에 집중한다.
  - 질의당 필요한 데이터만 불러와 컨텍스트를 최소화한다.
  - 온디맨드 실행 기준으로 운영하고, 불필요한 상시 배치 예측은 피한다.
- MVP 비용 방향
  - 조회형은 저비용 구조
  - 예측형은 조회형보다 비용이 높지만, 호출 수를 제한해 관리 가능한 수준으로 운영한다.
